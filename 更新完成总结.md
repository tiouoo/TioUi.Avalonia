## 🔍 关键特性

### MessageCloseReason 枚举

```csharp
public enum MessageCloseReason
{
    Timeout,      // 超时自动关闭
    UserAction,   // 用户手动关闭
    Displaced     // 被新消息替换
}
```

### 新增的管理器方法

```csharp
// INotificationManager / IToastManager
void Close(INotification notification);  // 关闭特定通知
void CloseAll();                         // 关闭所有通知
```

### 更新的回调签名

```csharp
// 旧签名
Action? OnClose

// 新签名
Action<MessageCloseReason>? OnClose
```

## 📊 与上游的差异

### 保留的本地特性

本项目保留了以下与上游不同的特性：

1. **多种关闭方法**
   - `Close(reason)` - 关闭并调用 OnRemove
   - `CloseWithoutRemovingFromList(reason)` - 仅关闭，不从列表移除
   - `RemoveAndDelete(reason)` - 移除并删除

2. **命名约定**
   - 使用 `Tio` 前缀（如 `TioNotificationManager`）
   - 使用 `TioMessageCard` 作为基类

3. **Toast 特殊行为**
   - Toast 超时使用 `CloseWithoutRemovingFromList()`
   - 保持本地的特殊关闭逻辑

所有这些本地特性都已更新以支持 `MessageCloseReason` 参数。

## 💡 代码示例

### 使用新功能

```csharp
// 1. 使用关闭原因回调
var notification = new Notification(
    "标题",
    "内容",
    NotificationType.Information,
    TimeSpan.FromSeconds(3),
    onClose: reason =>
    {
        Console.WriteLine($"关闭原因: {reason}");
        // Timeout, UserAction, 或 Displaced
    }
);
notificationManager.Show(notification);

// 2. 关闭特定通知
notificationManager.Close(notification);

// 3. 关闭所有通知
notificationManager.CloseAll();

// 4. Toast 同样支持
var toast = new Toast(
    "消息",
    NotificationType.Success,
    TimeSpan.FromSeconds(2),
    onClose: reason => Console.WriteLine($"Toast 关闭: {reason}")
);
toastManager.Show(toast);
toastManager.Close(toast);
toastManager.CloseAll();
```
